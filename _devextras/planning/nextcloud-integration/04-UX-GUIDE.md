# Step 4: UX Guide

## Design Principles
1. **Minimal clicks**: Most actions should be 1-2 clicks max.
2. **Context preservation**: User shouldn't lose their place in Nextcloud.
3. **Progressive disclosure**: Simple first, advanced options available.
4. **Escape hatch**: Always offer "Open in Synaplan" for power users.

## "Open in Synaplan" Behavior

Every chat panel and modal should have an escape hatch:

| Context | Link Behavior |
|---------|---------------|
| Settings page | Opens Synaplan dashboard |
| Summary modal | Opens Synaplan with file (if uploaded) |
| Document Chat | Opens Synaplan chat with file context |
| Research Chat | Opens Synaplan Research chat |

**URL Pattern:** `{synaplan_url}/chat?context=nextcloud&file={filename}`

This allows Synaplan to potentially receive the context and continue the conversation there.
